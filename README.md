# LiPicons

> Hand-drawn icon system for Hospital Information Systems
> **1,149** icons · **115** categories · **6** variants · by **imagiQa**

**🎨 Browse the catalogue: [arunshivab.github.io/lipicons](https://arunshivab.github.io/lipicons/)**

A free, hand-crafted icon family designed for HIS, EMR, RIS, LIS, pharmacy,
laboratory, billing, and adjacent healthcare software. Every icon is drawn
on a 24×24 grid with a 1.75 px stroke; `currentColor` lets CSS / parent
context decide the tint.

## Variants

| Variant   | Stroke | Notes |
| --------- | ------ | --------------------------------- |
| Regular   | 1.75   | The everyday weight |
| Bold      | 2.25   | Emphasis, primary CTAs |
| Light     | 1.25   | Dense layouts, secondary nav |
| Thin      | 1.00   | Marketing, hero illustrations |
| Fill      | —      | Solid silhouette |
| Duotone   | 1.5    | Stroke + 22% currentColor fill |

(Colour variant deferred to v1.1.)

## Quick start

### Fetch a sprite + reference by id (recommended)

Cross-origin `<use href="cdn-url...">` is blocked by most browsers — the
reliable pattern is to fetch the sprite via JS and inline it:

```html
<script>
  fetch('https://cdn.jsdelivr.net/gh/arunshivab/lipicons@v1.0.1/dist/lipicons.duotone.svg')
    .then(r => r.text())
    .then(s => document.body.insertAdjacentHTML('afterbegin', s));
</script>

<svg width="24" height="24"><use href="#heart-pulse"/></svg>
```

### Single-file direct from CDN

```html
<img
  src="https://cdn.jsdelivr.net/gh/arunshivab/lipicons@v1.0.1/dist/svg/duotone/heart-pulse.svg"
  width="24" height="24" />
```

*(Per-icon SVGs are generated by `npm run build` — run that locally first
to enable the URL pattern above.)*

### Just the JSON manifest

```
https://cdn.jsdelivr.net/gh/arunshivab/lipicons@v1.0.1/dist/json/icons.json
```

Every icon as `{ cat, keywords, paths }` — pull and render in any framework.

## Build from source

```bash
npm run build           # regenerates dist/svg/{variant}/{name}.svg
npm run build:sprite    # regenerates dist/lipicons*.svg sprites
```

Zero npm dependencies — uses only Node's built-in `fs`, `path`, `vm` modules.
Requires Node ≥ 18.
